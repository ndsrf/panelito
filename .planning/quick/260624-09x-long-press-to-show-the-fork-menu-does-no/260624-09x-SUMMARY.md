## What Was Built

### Task 1 — `useLongPress` rework

**Problem:** The hook's `onTouchMove` unconditionally cancelled the timer on any movement. Native text selection always introduces a few pixels of micro-movement immediately after the OS registers a long-press candidate, so the gesture was always killed. Additionally, the hook never called `preventDefault`, so the native selection callout and magnifier appeared freely.

**Fix:**
- Added `startCoordsRef` to track the touchstart position.
- `onTouchMove` now only cancels if `sqrt(dx^2 + dy^2) > MOVE_THRESHOLD` (10px). Sub-threshold jitter is ignored.
- `onTouchStart` calls `e.preventDefault()` when `e.cancelable` is true, blocking the native callout/magnifier from initializing.
- On long-press trigger, `window.getSelection()?.removeAllRanges()` clears any residual highlight.
- Mouse handlers remain unchanged — no `preventDefault`, preserving desktop click/double-tap.
- `useDoubleTap` is byte-for-byte unchanged.

### Task 2 — Bubble `select-none` + `.bubble-pressable` class

**Problem:** Both bubble `<div>` containers used `select-text` (enabling native selection) and had no WebKit touch properties to suppress the iOS long-press system menu or Android tap highlight.

**Fix:**
- Replaced `select-text` with `select-none` on both AI bubble and human bubble container divs.
- Added `bubble-pressable` class to both containers.
- Defined `.bubble-pressable` in `globals.css`:
  - `-webkit-user-select: none; user-select: none` — cross-browser selection suppression
  - `-webkit-touch-callout: none` — removes iOS long-press system callout menu
  - `-webkit-tap-highlight-color: transparent` — removes Android grey tap flash
- Avatars, overlays, and the restore-chart button are unaffected.
