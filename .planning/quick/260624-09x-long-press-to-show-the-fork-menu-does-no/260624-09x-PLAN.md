<task type="auto">
  <name>Task 1: Make useLongPress suppress native selection and tolerate micro-movement</name>
  <files>apps/web/hooks/use-long-press.ts</files>
  <action>
Rework useLongPress so a touch long-press wins over native OS text selection:
- Track the starting touch coordinates on onTouchStart. In onTouchMove, only cancel the timer if the finger has moved beyond a small tolerance (use a 10px movement threshold computed from the touch clientX/clientY delta). Movement under the threshold must NOT cancel — this is what currently kills the gesture.
- On the touchstart that begins a press, call e.preventDefault() only when the event is cancelable, to suppress the native selection/callout from initiating. Do NOT preventDefault on mouse handlers (would break desktop click/double-tap).
- When the 500ms timer fires (long-press confirmed), in addition to navigator.vibrate?.(10) and handler(), clear any in-progress native selection via window.getSelection()?.removeAllRanges() so no stray highlight remains.
- Keep the existing public signature and all returned handler keys (onTouchStart, onTouchEnd, onTouchMove, onTouchCancel, onMouseDown, onMouseUp, onMouseLeave). useDoubleTap stays byte-for-byte unchanged.
Note: the handlers must receive the real React event objects (the current implementation calls start()/cancel() with no event — update those arrow functions to accept and use the event for coordinates and preventDefault, per LAYOUT-06).
  </action>
