<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| any adapter → route | tool_use input is untrusted model output; must be validated before reaching panelStore |
| browser /settings → keys API | creator submits provider + key; activation changes server behavior for all sessions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | Tampering | Malformed/partial render_panel payload from any provider | mitigate | PanelWidgetSchema.safeParse gate added in /invoke before panel_update SSE; invalid dropped silently (AI-05) — closes critical failure modes 1 & 4 |
| T-04-13 | Information Disclosure | Decrypted key in /invoke | accept | Key decrypted server-side only, held in-memory for the request, never written to any SSE event (T-02-06) — unchanged from Phase 2 |
| T-04-14 | Denial of Service | Adapter hangs mid-stream | mitigate | Adapters yield done in finally (Plans 01-02); existing SSE error handler closes the stream; participants are not frozen (AI-SPEC done-event timeout is a Phase 5 addition, noted) |
| T-04-15 | Information Disclosure | Provider failure error surfaced in chat (D-07) | mitigate | On stream error the route emits a generic `error` SSE (`stream_failed`) — no provider-specific detail leaked to participants |
</threat_model>
