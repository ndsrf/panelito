Current AIStreamEvent lives in packages/types/src/panel.ts (lines 64-67) AND a parallel
definition lives in apps/api/src/lib/ai-provider.ts (lines 22-37). These MUST be unified.

The panel.ts AIStreamEvent uses `panel_update` (frontend SSE shape):
  | { type: 'text_delta'; text: string }
  | { type: 'panel_update'; payload: PanelWidget }
  | { type: 'done' }

The ai-provider.ts AIStreamEvent uses `tool_use` (adapter shape):
  TextDeltaEvent | ToolUseEvent | DoneEvent  where ToolUseEvent = { type:'tool_use'; name:string; input:unknown }

These are DIFFERENT event vocabularies. The adapter emits `tool_use`; the route translates
`tool_use` → `panel_update` SSE. Do NOT merge them into one type. The NEW canonical adapter-side
AIStreamEvent (text_delta | tool_use | done) belongs in packages/types/src/ai.ts. The frontend
`panel_update` union stays in panel.ts but should be RENAMED to avoid the name clash — keep the
adapter-side `AIStreamEvent` as the exported one from ai.ts; rename the panel.ts one to
`FrontendStreamEvent` and update its single re-export in index.ts (line 29).

Current renderPanelTool (apps/api/src/lib/ai-provider.ts lines 56-79) is typed Anthropic.Tool
with `input_schema`. New shape uses `parameters`.
