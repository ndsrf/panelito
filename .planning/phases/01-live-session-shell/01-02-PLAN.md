<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Next.js middleware | Cookie-based session; middleware refreshes the JWT on each request via `@supabase/ssr` |
| browser → Supabase Auth | OAuth handshake via Supabase GoTrue; PKCE flow is the default in `@supabase/ssr` |
| Google → Supabase callback | Google redirects to `http://127.0.0.1:54321/auth/v1/callback` (local) — auth code is exchanged for the Supabase session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Forged OAuth callback (attacker submits a fake code) | mitigate | Use `@supabase/ssr` canonical `exchangeCodeForSession` — Supabase validates the code against GoTrue's internal PKCE/state store; forged codes are rejected. Do not hand-roll the exchange. |
| T-02-02 | Tampering | Session cookie tampering | mitigate | Supabase cookies are signed by GoTrue; `@supabase/ssr` validates signatures on every read; tampered cookies are treated as unauthenticated. HttpOnly + Secure + SameSite=Lax flags are the `@supabase/ssr` defaults. |
| T-02-03 | Information Disclosure | OAuth callback error leaks auth code in referrer | mitigate | Callback handler strips `?code` from URL via Next.js `redirect()` to `/` (or `/auth/sign-in?error=oauth_failed` with no code); no `Referer` leak because we 307 to a clean URL. |
| T-02-04 | Repudiation | Sign-in events not audited | accept | Supabase Auth logs every sign-in in `auth.audit_log_entries` automatically; no extra logging needed in Phase 1. |
| T-02-05 | Elevation of Privilege | Middleware fails open if Supabase call throws | mitigate | Middleware wraps the auth check in try/catch; on error, treats user as unauthenticated and redirects to sign-in (fail closed). |
| T-02-06 | Information Disclosure | Server-only code (`lib/auth.ts`) accidentally imported in a client component | mitigate | `import 'server-only'` directive on line 1 — Next.js throws a build error if any client component imports this file. |
| T-02-SC | Tampering | `@playwright/test` install | mitigate | Playwright is the de-facto Microsoft-maintained E2E runner (github.com/microsoft/playwright); pinned to `1.49.x`; verified against the npm registry by the Plan 01 audit. |
</threat_model>
