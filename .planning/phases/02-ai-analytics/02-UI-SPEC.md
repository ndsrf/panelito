| Token | Value | Usage |
|-------|-------|-------|

Exceptions (carried forward from Phase 1):
- Touch targets: minimum **44px** tall on all interactive elements (reaction buttons, persona card tap area). (declared exception: WCAG 2.5.5 / iOS HIG minimum touch target requirement, carried forward from Phase 1)

Phase 2 additions:
- Reaction badge height: **24px** (emoji 14px + count 13px in a pill; not a touch target — display only).
- Streaming indicator dot: **8px** diameter.
- Persona badge height: **20px** (compact inline label in bubble header, not a touch target).

## Typography

Inherited from Phase 1 — no new sizes or weights. Declared type scale applies to all Phase 2 surfaces.

| Role | Size | Weight | Line Height | Phase 2 Usage |
|------|------|--------|-------------|---------------|

- Recharts tooltip: Body (15px Regular), `--card` surface.
- Bento grid card title: Label (13px Regular, Muted Foreground) above; Heading (20px Semibold, Foreground) for value or concept name below.

### Surface 1: Analytics Panel — Dynamic Widget Zone

**Container:** Inside `<AnalyticsPanelErrorBoundary>`, replacing the Phase 1 empty states when `widgetType !== null`.
**Layout:** Full panel area (`height: calc(var(--app-height) * 0.40)`), `overflow: hidden`. Widget fills the area minus any internal padding.
**Background:** `--card` (Zinc 900) — same as Phase 1 panel shell.

#### Panel Header Strip (PANEL-03)

A slim header at the top of the panel, height **36px — approved exception for panel header strip: 13px label + 12px vertical padding (top + bottom) × 2 = 24px + 1px border × 2 = 26px minimum; rounded to 36px for comfortable visual breathing room with branch badge.**, `flex-shrink: 0`.
Background: transparent (inherits panel `--card`). Bottom border: 1px `--border`.

Contents (flex row, 16px left padding, 8px right padding, vertically centered):
- Branch badge: `<Badge>` pill variant, Label (13px Regular). Background: active branch color at 20% opacity. Border: 1px solid active branch color. Text: branch color at light variant (e.g. Indigo 300 for Main branch). Left dot: 6px circle filled with branch color. Content: "Main" (Phase 2 only has Main branch). Touch target: none — display badge only.
- Right side: muted label "Analizando..." shown only while AI is streaming to the panel, with Indigo 500 `animate-pulse` dot. Hidden at rest.

#### Widget Zone

Height: `calc(100% - 36px)` (panel height minus header strip). `overflow: hidden`. `padding: 16px`.

The widget registry renders one of 4 widget types. Each widget fills this zone entirely.

**Framer Motion wrapper (D-08):**
All 4 widget types are wrapped in a single `<AnimatePresence mode="wait">` at the widget zone level. Transitions apply both to in-branch AI updates (new widget data) and branch switches (different snapshot).

Animation spec — widget transition:
```
initial:  { opacity: 0, scale: 0.97, filter: "blur(4px)" }
animate:  { opacity: 1, scale: 1,    filter: "blur(0px)" }
exit:     { opacity: 0, scale: 1.02, filter: "blur(4px)" }
transition: { duration: 0.28, ease: [0.25, 0.46, 0.45, 0.94] }
```
The scale + blur combination produces the "morphing and evolving" feel per D-08 specifics.
Use a `key` prop equal to `${widgetType}-${branchId}` to force unmount/remount on both widget type change AND branch switch.

#### Widget 1: Bento Grid (key concept cards)

Layout: CSS grid, `grid-cols-2` on mobile, `grid-cols-3` on desktop (≥768px breakpoint). Gap: 8px. `overflow-y: auto` within the widget zone.

Each bento card (`<Card>` — shadcn):
- Background: `--muted` (Zinc 800).
- Border: 1px `--border`.
- Border-radius: 8px (shadcn `--radius` = 0.5rem).
- Padding: 12px (3px deviation from 8pt scale — approved exception for compact card grid).
- Content (flex column, 4px gap):
  - Category label: Label (13px Regular, Muted Foreground). Single line, truncates.
  - Concept name: Heading (20px Semibold, Foreground). Max 2 lines, `line-clamp-2`.
  - Optional: relevance score pill (Label 13px Regular, Muted Foreground) — right-aligned small badge. Background `--card`. Width auto.

Empty card placeholder (when AI sends fewer than grid capacity):
- Same card shell, `--muted` background at 50% opacity. No content. Not interactive.

#### Widget 2: Radar Chart (multi-axis scoring)

Recharts `<RadarChart>` centered in widget zone.
- Chart container: 100% width × 100% height (`<ResponsiveContainer>`).
- `<PolarGrid>` stroke: `--border` (`#3f3f46`).
- `<PolarAngleAxis>` tick: Label style (11px, `--muted-foreground`).
- `<PolarRadiusAxis>` hidden (no numeric rings — too noisy in compact space).
- `<Radar>` fill: Indigo 500 at 20% opacity. Stroke: Indigo 500. Stroke-width: 2.
- `<Tooltip>` content: custom component, `--card` background, `--border` border, Body (15px Regular).
- No `<Legend>` — axis labels on `PolarAngleAxis` are sufficient.

#### Widget 3: Scatter Plot (consensus/impact positioning)

Recharts `<ScatterChart>` with two axes: X = "Consenso" (0–100), Y = "Impacto" (0–100).
- Chart container: `<ResponsiveContainer>` 100% × 100%.
- `<CartesianGrid>` stroke: `--border` at 40% opacity, strokeDasharray="3 3".
- `<XAxis>` label: "Consenso", Label (11px, `--muted-foreground`). Domain [0, 100].
- `<YAxis>` label: "Impacto", Label (11px, `--muted-foreground`), rotated. Domain [0, 100].
- `<Scatter>` fill: Indigo 500. `shape`: custom circle component, radius 6px, with a `title` attribute for accessibility.
- Each point may have an optional text label (concept name, max 20 chars, truncated) rendered via a custom dot component at 13px Muted Foreground, offset 8px above the dot.
- `<Tooltip>` content: custom, `--card` background. Shows "Consenso: X | Impacto: Y | [concept]".
- No `<Legend>`.

#### Widget 4: Pie Chart (proportional breakdown)

Recharts `<PieChart>` centered in widget zone.
- Chart container: `<ResponsiveContainer>` 100% × 100%.
- `<Pie>` innerRadius 40% (donut) outerRadius 70%. `paddingAngle: 3`.
- Segment fill order: Indigo 500, Violet 400, Sky 400, Emerald 400, Amber 400, then repeat from index 0 for >5 segments.
- `<Tooltip>` content: custom, `--card` background. Shows "[label]: X%".
- `<Legend>` rendered below chart (vertical list, Label 13px Regular, `--muted-foreground`). Max 6 items; overflow truncated.
- Stroke between segments: `--background` (`#09090b`), width 2px — creates separation on dark bg.

### Surface 2: AI Message Bubble Variant (PERSONA-03)

**Extends:** existing `MessageBubble` component. Phase 2 adds an `isAI` prop and renders a distinct variant when true.

**Anatomy differences from human message bubble:**

Avatar:
- Shape: same 32px circle.
- Background: Indigo 500 at 20% opacity (`rgba(99, 102, 241, 0.20)`).
- Border: 1px solid Indigo 500.
- Icon: Lucide `Bot` (16px, Indigo 400 `#818cf8`) centered — NOT an initials letter.
- `aria-hidden="true"`.

Bubble header row (same flex row as human: author name + timestamp):
- Author name: "Analista Científico" — Body (15px Regular, Foreground). NOT the display_name from DB.
- Persona badge: inline after the author name, 4px gap.
  - Content: "Analista Científico" label (already shown as author name — so the badge shows only the icon for compactness).
  - Badge: `<Badge>` pill, 20px height. Background: Indigo 500 at 12% opacity. Border: 1px solid Indigo 500 at 30% opacity. Contains: Lucide `FlaskConical` icon (10px, Indigo 300 `#a5b4fc`) + "Analista" text (Label 13px Regular, Indigo 300). 4px internal gap.
  - `aria-label="Analista Científico — AI persona"`.
- Timestamp: same as human (Label 13px Regular, Muted Foreground).

Bubble content area:
- Background: `--card` (Zinc 900) — same as other-human bubbles.
- Left border accent: 2px solid Indigo 400 (`#818cf8`). `border-radius: 0 4px 4px 0`. Applied to content column.
- Streaming state: when AI is mid-stream, the bubble shows a blinking cursor `▋` after the last token (CSS `animate-pulse`, Indigo 500 color). The cursor disappears when stream ends.
- Max-width: `max-w-[90%]` (wider than human max-w-[80%] — AI responses are typically longer and deserve more horizontal space).

Reaction badges on AI bubbles: same spec as human (see Surface 3). AI bubbles are reactable.

Streaming indicator (AI-03):
- Rendered inside the bubble content area, before text appears.
- Three pulsing dots: 6px circles, Indigo 500, `animate-bounce` with staggered delays (0ms, 150ms, 300ms).
- Visible only during the pre-first-token wait period. Replaced by streaming text once first token arrives.
- `aria-label="Analista está escribiendo..."`, `role="status"`.

### Surface 3: Reaction Badges on MessageBubble (REACT-01–05)

**Placement:** Below the message content bubble, within the content column. Not inside the bubble rectangle — outside it with 4px gap above (sm spacing).

**Layout:** Horizontal flex row, 4px gap between badges. `flex-wrap: wrap` if many reactions. Aligned `flex-start` (left).

**Individual reaction badge anatomy:**
- Container: pill shape, height 24px, horizontal padding 6px.
- Background: `--muted` (Zinc 800). Border: 1px `--border`.
- Border-radius: 999px (full pill).
- Content: `{emoji}` (14px, native emoji color) + space + `{count}` (Label 13px Regular, Muted Foreground). 2px gap between emoji and count (declared exception: 2px for emoji-to-count gap within 24px pill height — 4px would be disproportionate at this scale).
- Example: `🔥 2`

**Own-reaction state (when current user has applied this reaction):**
- Border: 1px solid Indigo 500 (`--primary`).
- Background: Indigo 500 at 10% opacity (`rgba(99, 102, 241, 0.10)`).
- Count text: Indigo 300 (`#a5b4fc`).

**Optimistic rendering (REACT-05):**
Badge appears immediately on tap. If server returns error, badge reverts (disappears or decrements count) with no animation — silent revert.

**Zero-count state:** Badges with count = 0 are not rendered. The badge appears only when at least 1 reaction exists.

**Touch target:** The badge itself is NOT a tap target in Phase 2 — it is display-only. Reactions are applied via QuickReactionPopover (double-tap on the bubble), not by tapping the badge.

**Accessibility:** `aria-label="{emoji} reaction, {count} times"`. Example: `aria-label="🔥 reaction, 2 times"`. Note: `aria-label` strings are in English for locale-neutral screen reader compatibility. All visible copy remains Spanish.

### Surface 6: Chat Input Soft-Lock Indicator (AI-07, D-04)

**Extends:** `InputBox` component. The lock indicator is already scaffolded — Phase 2 activates it.

**Lock state trigger:** Supabase Presence/Realtime channel broadcasts `{ type: 'ai_streaming', streaming: true }` when an AI response is in flight. All participants receive this signal.

**Visual changes during AI stream lock (in addition to Phase 1 placeholder text change):**

Send button: disabled, `--muted` fill (inherited from Phase 1 locked state spec).

Textarea opacity: 0.5 (inherited from Phase 1 locked state spec).

**Unlock:** when AI stream `done` event fires and `streaming: false` presence update is broadcast, all lock states revert to idle. Transition: no animation (instant revert).

## Framer Motion Animation Specs (consolidated)

| Surface | Animation | Spec |
|---------|-----------|------|
| Widget zone — type change or branch switch (D-08) | Cross-fade + scale + blur morph | `initial: {opacity:0, scale:0.97, filter:"blur(4px)"}` → `animate: {opacity:1, scale:1, filter:"blur(0px)"}` → `exit: {opacity:0, scale:1.02, filter:"blur(4px)"}`. Duration 280ms, ease `[0.25, 0.46, 0.45, 0.94]`. Key: `${widgetType}-${branchId}`. |
| AI bubble appear (new message from stream) | Fade + slide up | `initial: {opacity:0, y:6}` → `animate: {opacity:1, y:0}`. Duration 200ms, ease "easeOut". Identical to human message appearance. |
| Reaction badge appear (optimistic) | Scale + fade | `initial: {scale:0.7, opacity:0}` → `animate: {scale:1, opacity:1}`. Duration 120ms, spring stiffness 400. |
| Reaction badge revert (server error) | Fade out | `exit: {opacity:0, scale:0.7}`. Duration 100ms. |
| Streaming dots (input + bubble) | Bounce stagger | CSS `animate-bounce`, Tailwind. No Framer needed. |
| Persona drawer slide in | Slide right (desktop) | shadcn `<Sheet>` Radix handles — Framer not needed. |

## Error States (Phase 2)

### Widget Schema Validation Failure (AI-05)

The `ui_mutation_block` fails Zod validation. The panel is NOT updated — last stable state is preserved (no error shown to user). No widget is rendered or removed. This is a silent discard per D-05.

### Widget Render Error (caught by error boundary)

If a rendered widget component throws during render (Recharts crash on unexpected but schema-valid data), the error boundary shows the Phase 1 fallback card:
- Icon: Lucide `AlertTriangle`, 24px, Destructive.
- Heading (20px Semibold, Destructive): "Error de visualización"
- Body (15px Regular, Muted Foreground): "El Analista está recalculando. Tu chat no se ve afectado."

### AI Stream Error (SSE connection drops mid-stream)

The incomplete AI message bubble shows the partial text already rendered plus an inline error note appended at the end of the bubble content:
- 8px gap above error note.
- Lucide `AlertCircle` (12px, Destructive) + Label (13px Regular, Muted Foreground): "Respuesta interrumpida. Vuelve a intentarlo."
- NOT a toast — inline in the bubble only.

### Reaction POST failure

Silent badge revert — no toast, no error message. The reaction was optimistic and silently removed. User may retry by double-tapping again.

### Persona toggle POST failure

Toast (bottom-center, `--card` surface, 4px left border Destructive):
- "No se pudo cambiar el analista. Inténtalo de nuevo."
- Auto-dismiss 4s.
