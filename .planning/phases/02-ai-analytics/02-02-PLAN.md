## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /invoke | Untrusted POST body (userMessage, anyoneTyping) crosses into the AI invocation |
| client → /reactions | Untrusted reaction payload crosses into Postgres + AI trigger decision |
| Anthropic key → server memory | Plaintext key decrypted server-side for the Claude call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Elevation of Privilege | client self-authorizing AI calls | mitigate | Server gates invoke on session ownership (creator_id === user.id → else 403) and on active_personas non-empty; the client cannot force an AI response when the persona is off |
| T-02-06 | Information Disclosure | API key leaking via SSE | mitigate | Key decrypted server-side only, used to construct AnthropicAdapter; never written to any SSE event or response body (AI-02) |
| T-02-07 | Denial of Service | reaction-spam → runaway AI cost | mitigate | checkCap() enforced before every invoke (429 if cap reached); reactionRateLimit 60/min/user; cap increments only on completed streams |
| T-02-08 | Tampering | path_id leakage across branches | mitigate | Message query enforces .eq('path_id', 'main') (AI-06); Phase 3 sibling content can never enter context |
| T-02-09 | Spoofing | reaction INSERT as another user | mitigate | RLS WITH CHECK (auth.uid() = author_id) from Plan 01 migration |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan; all dependencies already installed and slopcheck-approved (RESEARCH Package Legitimacy Audit) |
</threat_model>
