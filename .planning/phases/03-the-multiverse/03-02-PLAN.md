# Plan: Navigation & State

Update the frontend state management to handle multiple branches and refactor the `BranchNavigator` to support dynamic switching, horizontal scrolling, and auto-centering.

## Requirements
- **BRANCH-03**: The Branch Navigator shows all open branches; tapping switches the active branch and triggers panel re-render.
- **BRANCH-06**: Switching branches immediately re-renders the analytics panel to the latest snapshot.

## User Decisions
- **D-14**: Horizontal Smart Scroll with auto-centering of the active branch chip.

## Tasks

### 1. State Store Update
<task id="store-update">

- `SessionStore` tracks `branches: Branch[]`.
